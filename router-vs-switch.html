Router vs Switch ‚Äî What's the Difference?

Both devices sit in your network rack and have cables plugging into them ‚Äî but they do completely different jobs at completely different layers of the OSI model. Here's how to tell them apart for good.

The one-sentence version

A switch connects devices within the same network and forwards traffic using MAC addresses. A router connects separate networks together and forwards traffic using IP addresses. That distinction ‚Äî MAC vs IP, same network vs different networks ‚Äî is the core of what the A+ exam tests.

How they fit together in a real network

Routers and switches aren't competitors ‚Äî they're partners. In almost every real network, both are present and each handles a different job. The router sits at the edge, facing the internet. The switch sits inside, connecting everything on the local network.

‚ö° A+ Exam ‚Äî The Key Memory Hook

Switch = Layer 2 = MAC addresses = same network.

Router = Layer 3 = IP addresses = between networks.

If the exam asks "what device separates broadcast domains?" ‚Äî the answer is router. If it asks "what device uses a MAC address table?" ‚Äî the answer is switch. If it asks "what is the default gateway?" ‚Äî that's the router's IP address on the local network.

Hub vs switch ‚Äî why it matters

üì° Don't confuse switches with hubs

A hub is an older Layer 1 device that blindly sends every incoming frame out to every single port ‚Äî it has no intelligence. Every device on a hub receives every frame, whether it's meant for them or not. This wastes bandwidth and creates collisions.

A switch is smarter ‚Äî it learns which device is on which port (via the MAC address table) and sends frames only to the correct port. This is called unicast forwarding and it's why switches replaced hubs. On the A+ exam, hubs still appear as a concept ‚Äî just remember: hub = Layer 1, broadcasts to all; switch = Layer 2, sends only to the destination.

Office with 40 devicesBoth needed
A 24-port switch connects all 40 devices on the local network. A single router connects the switch (and therefore all devices) to the internet and acts as the default gateway. Both devices are essential ‚Äî neither can replace the other.
üîÄ
Layer 3 switch ‚Äî when things overlapAdvanced
Some enterprise switches are "Layer 3 switches" ‚Äî they can do basic IP routing in addition to MAC-based switching. These blur the line between routers and switches. On the A+ exam, assume a switch is Layer 2 unless specifically told otherwise.

Key Takeaways

Router = Layer 3, IP addresses, connects different networks ‚Äî it's the device that gets you to the internet
Switch = Layer 2, MAC addresses, connects devices on the same network ‚Äî it's what your PC, printer, and server all plug into
The router IS the default gateway ‚Äî when traffic needs to leave the local network, it goes to the router first
Switches use a MAC address table (CAM table) to forward frames only to the correct port ‚Äî far more efficient than a hub
Hub = Layer 1, sends to all ports (dumb). Switch = Layer 2, sends only to destination (smart). Hubs are obsolete but still appear on the A+ exam.
Routers separate broadcast domains ‚Äî switches do not (unless VLANs are configured)
In most networks, both devices work together: switch handles local traffic, router handles internet traffic
